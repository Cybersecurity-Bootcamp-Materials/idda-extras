📚 Theory

What Is Malware?

Malicious software designed to disrupt, damage, or gain unauthorized access to systems

🔤
Broad Category
Malware is an umbrella term — ransomware, viruses, worms, trojans, spyware, adware, and rootkits are all types of malware
🔄
Intent Matters
All malware is intentionally written to cause harm, spy, extort, or enable unauthorized access
🌎
Universal Threat
Targets individuals, corporations, hospitals, governments — no organization is too small or too large
📈
Growing Problem
Over 450,000 new malware samples are detected every day. Cost of cybercrime expected to reach $10.5 trillion annually by 2025
Key Distinction: Not all malware is equal. Understanding how each type behaves determines which defenses are most effective against it.

📚 Theory

Ransomware

Encrypts your files, demands payment for the key

🔒 Locker Ransomware

  • Locks the entire device — mouse, keyboard, screen
  • Does not encrypt files directly
  • Basic computer functions are disrupted
  • Example: early Android screen-lockers

🔐 Crypto Ransomware

  • Encrypts files and documents specifically
  • Device remains usable — just data is locked
  • Much more dangerous and common today
  • Examples: WannaCry, REvil, LockBit, Conti

Why Cryptocurrency?

Attackers demand payment in Bitcoin or Monero because:

  • Transactions are pseudonymous / anonymous (Monero)
  • No chargebacks — payment is irreversible
  • Cross-border payments avoid banking oversight
  • Funds can be quickly laundered through mixers
No Guarantee: Even after paying the ransom, ~20% of victims never receive a working decryption key. Paying also incentivizes future attacks.
📚 Theory

Ransomware Attack Life Cycle

Eight stages from initial delivery to post-attack

#StageWhat Happens
1DeliveryPhishing email, drive-by download, or RDP brute force
2ExecutionVictim opens attachment / visits compromised site; payload runs
3PersistenceMalware installs itself in startup locations to survive reboots
4EncryptionFiles are encrypted using strong asymmetric keys (RSA + AES)
5Ransom NoteVictim sees a README or pop-up with payment instructions
6PaymentVictim uses Tor browser to contact attacker, pays in crypto
7Decryption KeyAttacker (maybe) provides decryption tool
8Post-AttackData may be leaked (double extortion); reinfection risk remains
Double Extortion: Modern ransomware groups (LockBit, Cl0p) exfiltrate data before encrypting it. They threaten to publish sensitive data if the ransom isn't paid — even if you restore from backups.

📚 Theory

Computer Virus

Self-replicating malware that requires a host to spread

Five Defining Characteristics

🔄
Replication
Attaches a copy of itself to other executable files or documents
📤
Propagation
Spreads when infected files are shared via email, USB, or network shares
💨
Activation
Triggers its payload under specific conditions (date, event, or action)
🕵
Concealment
Encrypts or obfuscates its code to evade signature-based detection
💥
Payload
Executes damage: deleting files, corrupting data, or installing more malware

Common Virus Types

  • File Virus — infects executable files (.exe, .com)
  • Macro Virus — hides in Office documents (Word macros)
  • Boot Sector Virus — infects the master boot record; activates before the OS loads
  • Browser / Web Scripting Virus — embedded in malicious web pages
📚 Theory

Detecting Computer Viruses

Three detection approaches — each with strengths and limitations

📑 Signature-Based

Matches file patterns against a database of known malware signatures.

Pro Fast, accurate for known threats

Con Useless against new / polymorphic viruses

📈 Behavior-Based

Monitors running processes for suspicious activity (file writes, registry changes).

Pro Can catch zero-day threats

Con Higher false positive rate

⛅ Cloud-Based / Heuristic

Sends suspect files to cloud for analysis using ML models and sandboxing.

Pro Always up to date; no local DB needed

Con Requires internet; privacy concerns

📊 Prevention Summary

  • Keep OS and apps patched
  • Use reputable AV software
  • Never open unexpected attachments
  • Avoid pirated / cracked software
  • Scan removable media before use
📚 Theory

Computer Worm

Self-replicating, network-aware — no host file required

What Makes Worms Different?

Unlike viruses, worms don't need a user to share an infected file. They autonomously scan networks, find vulnerable machines, and copy themselves — often within minutes of initial infection.

Types of Worms

TypeSpread MethodFamous Example
Internet WormExploits network service vulnerabilitiesWannaCry (SMB / EternalBlue)
Email WormSends itself to all contacts in address bookMyDoom (2004)
Instant Message WormSends malicious links via IM platformsKoobface (Facebook)
File-Sharing WormDisguised as popular files on P2P networksSality
WannaCry (2017): Infected 230,000+ computers in 150 countries in one day by exploiting an unpatched Windows SMB vulnerability. Caused $4–8 billion in damages. The NHS UK was severely disrupted.
📚 Theory

Virus vs Worm

Common confusion — here's how to tell them apart

CharacteristicVirusWorm
Needs host file?YesNo
Spreads without user action?NoYes
Exploits network vulnerabilities?RarelyCommonly
Speed of spreadSlow (requires sharing)Very fast (autonomous)
PayloadYes (data destruction, etc.)Optional (can carry payload)
Typical detectionFile scannerNetwork IDS / anomaly detection

Worm Prevention Checklist

  • Patch promptly — worms exploit known, unpatched vulnerabilities
  • Network segmentation — isolate critical systems to limit spread
  • Disable unused services — SMBv1, Telnet, etc.
  • IDS/IPS monitoring — detect anomalous port scanning behavior

📚 Theory

Trojan Horse

Deception is the weapon — malicious code disguised as something legitimate

Core Concept

Trojans rely entirely on social engineering. They don't replicate. Instead they convince users to voluntarily install them by pretending to be a useful app, game, update, or document.

Common Trojan Types

📄
Backdoor Trojan
Opens a remote access channel; attacker can execute commands, upload/download files
💰
Banking Trojan
Intercepts banking sessions, injects fake forms to harvest credentials (e.g. Zeus, Emotet)
👤
RAT (Remote Access Trojan)
Full remote control: webcam, keylogger, screen capture, file system access
▼
Downloader Trojan
Foothold only — downloads and installs additional malware (ransomware, RATs) after execution
🕵
Rootkit Trojan
Installs rootkits to hide attacker's presence and maintain persistent, stealthy access
🛡
Antivirus Fraud Trojan
Fakes AV alerts to scare users into "purchasing" a fix that installs more malware

📚 Theory

Adware

Advertising-supported software — intrusive, privacy-invasive, and often unwanted

💲 Legitimate Adware

User knowingly agrees to see ads in exchange for free software. Disclosed in Terms of Service. Can be uninstalled.

Low Risk

🔐 Unwanted Adware

Installed without informed consent, bundled with other software. Difficult to remove. Often includes spyware components that track browsing.

High Risk

Revenue Models Used by Adware Creators

ModelHow It Works
Pay Per Click (PPC)Adware generates fake clicks on ads, earning revenue per click even without user intent
Pay Per View (PPV)Ad impressions are sold — adware inflates view counts by force-loading ads
Data BrokeringBrowsing history and personal data is collected and sold to third-party data brokers
Browser HijackingDefault search engine replaced to redirect searches through revenue-generating engine

VirusTotal

Free online multi-engine malware scanner — a SOC analyst's daily tool

What It Does

VirusTotal aggregates 70+ antivirus engines and sandbox tools to analyze:

  • Files — upload any file up to 650 MB for instant multi-AV scan
  • URLs — check if a link leads to malicious content or phishing
  • Hashes — look up MD5 / SHA-1 / SHA-256 to find previous scan reports
  • IP addresses & domains — check reputation and associated malware

Key Features

⚡
Real-Time Detection
Results from 70+ engines in seconds; database updated continuously
👥
Community Comments
Security researchers add context, behavioral notes, and YARA rules
🔗
Relationship Graph
Visualize connections between files, domains, IPs, and threat actors
📊
Behavioral Analysis
Sandbox execution shows file writes, registry changes, and network calls
📚 Theory

Using VirusTotal Effectively

Practical workflow and important limitations to understand

Analyst Workflow

  1. Hash first — look up the SHA-256 hash before uploading; avoids sending sensitive files to a public platform
  2. Check detection ratio — "3/72 engines" vs "65/72 engines" tells very different stories
  3. Read engine names — reputable engines (Kaspersky, Microsoft, Sophos) flagging = high confidence
  4. Check the Behavior tab — sandbox report reveals what the file actually does
  5. Explore Relations — see what domains/IPs the file contacts, related hashes

Critical Limitations

False Positives: A file flagged by 1–2 obscure engines may be a false positive. Context matters — check which engines, how reputable, and what category they label it.
Privacy Warning: Any file uploaded to VirusTotal becomes available to VirusTotal's premium subscribers. Never upload confidential documents, source code with credentials, or PII. Hash lookup is the safer alternative.
Clean ≠ Safe: 0/72 detections doesn't guarantee safety. New zero-day malware or custom samples won't match any signatures yet.

📚 Theory

Endpoint Detection & Response (EDR)

Real-time visibility and response at the device level — beyond traditional AV

Why Traditional AV Isn't Enough

Classic antivirus matches known signatures. It misses:

  • Zero-day exploits (no signature exists yet)
  • Living-off-the-land attacks (using legitimate OS tools like PowerShell, WMI)
  • Fileless malware (runs entirely in memory)
  • Lateral movement after initial compromise

EDR Core Capabilities

📺
Continuous Recording
Records all endpoint activity — process creation, file writes, network connections, registry changes
📈
Behavioral Analysis
Flags deviations from normal behavior, not just known signatures
🔌
Endpoint Isolation
Can quarantine a compromised host from the network with one click to stop lateral movement
🔍
Threat Hunting
Analysts can query historical data to find threats that bypassed initial detection
📚 Theory

EDR vs SIEM

Complementary tools — not alternatives

AspectEDRSIEM
FocusIndividual endpoints (devices)Organization-wide log aggregation
Data SourceEndpoint agent telemetryLogs from all systems (network, apps, cloud)
ResponseCan isolate endpoint, kill process, quarantine fileAlert + workflow; response is manual
DetectionBehavioral + signatures at host levelRule-based correlation across data sources
Threat HuntingDeep endpoint forensicsBroad correlation across the environment
ExamplesCrowdStrike Falcon, SentinelOne, WazuhSplunk, Microsoft Sentinel, Wazuh
Together they're stronger: SIEM gives you the big picture across your entire environment. EDR gives you deep forensic detail and automated response at each machine. Wazuh uniquely combines both.

📚 Theory

EDR in the SOC Workflow

How SOC analysts use EDR day-to-day

Typical Alert Response Flow

  1. Alert fired — SIEM / EDR raises alert (e.g. suspicious PowerShell encoded command)
  2. Triage — analyst checks the host's process tree, parent/child relationships
  3. Enrich — look up file hashes on VirusTotal; check IOC feeds
  4. Contain — if malicious, isolate the host via EDR one-click isolation
  5. Investigate — timeline the attack using EDR historical data; identify patient zero
  6. Remediate — delete malicious files, restore from backup, patch vulnerability
  7. Document — write incident report; update detection rules to catch similar activity
Lab Context: In the Wazuh lab, you'll see this workflow in action — triggering alerts via test events, reading them in the Wazuh dashboard, and understanding how agent telemetry maps to security events.

EDR & Wazuh Quiz

Three questions — some advanced, think carefully

1. A malware sample runs entirely in memory and never writes to disk. Traditional AV won't detect it. Which EDR capability is best suited to catch this?
A File Integrity Monitoring
B Signature-based scanning
C Behavioral / anomaly detection
D Vulnerability scanning
Correct! Fileless malware has no file to scan. Behavioral analysis detects the suspicious actions it takes (injecting into processes, unusual network calls) regardless of how it runs.
Not quite. FIM and signatures need a file. Behavioral analysis monitors what a process *does*, not what it *is* — making it the right tool for fileless threats.
2. Wazuh's FIM module generates an alert. What does this mean?
A A firewall rule was triggered
B A monitored file or directory was created, modified, or deleted
C A new user account was created
D Suspicious network traffic was detected
Correct! FIM = File Integrity Monitoring. It watches critical system files (like /etc/passwd on Linux or Windows System32) and alerts on any unauthorized changes.
Not quite. FIM = File Integrity Monitoring. It detects unauthorized changes to critical files — a common indicator of malware persistence or attacker activity.
3. Why is endpoint isolation an important EDR feature during an active ransomware incident?
A It decrypts the affected files automatically
B It cuts off the infected machine from the network, preventing spread to other hosts
C It removes the ransomware from memory immediately
D It forces a reboot to clear the infection
Correct! Speed is everything with ransomware. Isolation severs the infected host from the network, stopping lateral movement and preventing encryption of shared drives or other machines.
Not quite. Isolation = cutting off network access. This stops the ransomware from spreading to shared drives, backup servers, or other machines while you respond.
